# Безопасность

**Русский** · [English](security.md)

Модель угроз, реализованные меры и hardening развёртывания **avia-bot** MVP. Обработка данных — [privacy_ru.md](privacy_ru.md).

---

## Область и допущения

| Аспект | Состояние MVP |
|--------|---------------|
| Развёртывание | Внутреннее демо / пилот в доверенной сети |
| Аутентификация | **Не реализована** — любой клиент может вызывать API |
| Авторизация | Нет разграничения по пользователям/ролям |
| Классификация данных | Операционные процедуры (в демо KB нет PII пассажиров) |
| LLM-провайдер | Внешний OpenAI-совместимый API (данные покидают периметр) |

Текущая сборка — **демонстрационная система**, не готовая к открытой сети.

## Модель угроз

| Угроза | Риск | Митигация (сейчас) | Пробел |
|--------|------|----------------------|--------|
| Prompt injection / jailbreak | Высокий | System prompt, разделители, regex guard | Не абсолютна; custom prompt отключает guards |
| Утечка через LLM | Средний | Ограничение scope в промптах | Текст пользователя уходит во внешний API |
| Несанкционированный доступ к API | Высокий | Нет в MVP | Изоляция сети; SSO/API keys на пилоте |
| Отравление KB | Средний | Контролируемый ingest | Нет подписанного контент-пайплайна |
| Перехват SSE | Низкий | Same-origin; HTTPS | Нет auth на события |
| Кража файла SQLite | Средний | Права ФС | Шифрование at rest в продакшене |

## Реализованные меры

### 1. Защита от prompt injection

Активна в режимах **LLM** и **RAG**, кроме **custom system prompt** (`llm_config.use_custom_prompt=true`).

| Слой | Модуль | Поведение |
|------|--------|-----------|
| System prompt | `app/llm/prompts.py` | Авиационный scope; отказ от jailbreak и off-topic |
| Усиление сообщения | `app/llm/prompt_guard.py` | Разделители `<<USER>>` … `<</USER>>`; санитизация |
| Pre-flight | `ChatService` | Regex для явных injection / off-topic |

Блокировка — вежливый отказ в тексте ассистента (`metadata.guard_refusal`), не HTTP-ошибка.

### 2. Grounding RAG

- Ответы в RAG на основе чанков + статические главы 00, 13.
- Trace panel показывает источники.
- Правила out-of-scope (гл. 13) в system prompt.

### 3. Дисклеймер

Глава 00 KB: бот — **инструмент поддержки**, не замена суждению сотрудника.

### 4. Обработка ошибок

Структурированный `error_code` — без stack trace клиенту.

### 5. CORS

`APP__CORS_ORIGINS` ограничивает origins браузера. Не защищает прямые вызовы API.

| # | Мера | Приоритет |

## LLM-провайдер

На каждый RAG-запрос провайдеру могут уходить:

- Вопрос и история чата
- Извлечённые чанки KB
- Статический текст политики (гл. 00, 13)
- Промежуточные промпты (HyDE, multi-query, rerank, decision tree)

**Рекомендации:** DPA и резидентность данных; не вставлять PII пассажиров; on-prem для регулируемых сред.

## Реагирование на инциденты

| Событие | Действие |
|---------|----------|
| Утечка API key | Ротация; проверка логов провайдера |
| Обход guard | Зафиксировать сообщение + trace; обновить паттерны |
| Опасный неверный ответ | Эскалация владельцу KB; отключение функции |
| Утечка БД | Ротация credentials; оценка чувствительности KB |
