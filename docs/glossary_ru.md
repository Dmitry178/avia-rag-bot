# Глоссарий

**Русский** · [English](glossary.md)

Термины документации продукта и разработки **avia-bot**.

---

## Продукт

| Термин | Определение |
|--------|-------------|
| **Avia-bot** | AI-ассистент для сотрудников аэропорта — ответы по внутренним регламентам (SOP, FAQ, сценарии). |
| **KB / База знаний** | Markdown-документ (`rag-document.md`) с операционными процедурами. |
| **SOP** | Standard Operating Procedure — пошаговые официальные инструкции (главы 01–12). |
| **FAQ** | Частые вопросы — краткие Q&A (глава 14 + блоки в главах). |
| **Decision tree** | Дерево решений для нестандартных ситуаций (глава 16). |
| **Scenario** | Нарративный пример ситуации и действий (глава 17). |
| **Карточка операционной процедуры** | UI-блок walkthrough decision tree над основным ответом RAG. |
| **Пилот** | Ограниченное внедрение с реальной KB и KPI (фаза 1 roadmap). |

---

## Режимы

| Термин | Определение |
|--------|-------------|
| **Режим LLM** | Прямой диалог с языковой моделью без поиска по KB. |
| **Режим RAG** | Retrieval-augmented generation — ответ на основе индексированных чанков. |
| **Guard** | Защита от prompt injection и off-topic запросов. |
| **Custom system prompt** | Пользовательский system prompt; отключает guards в LLM. |

---

## RAG pipeline

| Термин | Определение |
|--------|-------------|
| **Chunk / Чанк** | Текстовый сегмент для векторного поиска; SQLite + строка FAISS. |
| **Lane** | Корпус retrieval (`sop`, `faq`, `decision_tree`, `scenario`) со своей квотой top-K. |
| **Content type** | Категория чанка, фильтр lane. |
| **HyDE** | Hypothetical Document Embeddings — поиск по гипотетическому ответу LLM. |
| **Multi-Query** | Несколько вариантов запроса + RRF по lane. |
| **Query Rewriting** | Переписывание запроса с учётом истории чата. |
| **Rerank** | LLM-оценка кандидатов после векторного поиска. |
| **RRF** | Reciprocal Rank Fusion — слияние ранжированных списков. |
| **top_chunks** | Число чанков в контексте генерации (3–21). |
| **Trace** | Пошаговый лог RAG (SSE + metadata сообщения). |
| **Static context** | Главы 00 и 13 в system prompt без FAISS. |

## Данные и индексация

| Термин | Определение |
|--------|-------------|
| **ETL** | Парсинг markdown, эмбеддинг, запись SQLite + FAISS. |
| **Ingest** | Запуск ETL для построения или обновления индекса. |
| **Manifest** | Метаданные последней сборки индекса. |
| **FAISS** | Библиотека векторного поиска (`IndexFlatIP`). |
| **Инкрементальный ingest** | Re-embed только изменённых чанков. |
| **Rebuild** | Полный re-embed всех чанков. |
| **Checkpoint** | Прогресс ingest для resume после прерывания. |

## Разработка

| Термин | Определение |
|--------|-------------|
| **Monorepo** | Один репозиторий: `backend/` и `frontend/`. |
| **SSE** | Server-Sent Events — sideband для trace и async-уведомлений. |
| **client_id** | UUID браузера для связи SSE и POST сообщения. |
| **Soft delete** | Флаг `is_deleted`; запись скрыта из API. |
| **DBManager** | Фасад доступа к БД на запрос. |
