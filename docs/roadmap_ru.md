## Фаза 0 — Демо (текущая)

**Цель:** Доказать работоспособность RAG и сравнить методы retrieval для стейкхолдеров.

| Область | Статус |
|---------|--------|
| Multi-lane RAG (SOP, FAQ, decision trees, scenarios) | Готово |
| Query transform (HyDE, Multi-Query, Rewriting) + Rerank | Готово |
| Pipeline trace (SSE + UI) | Готово |
| Режим LLM + guards | Готово |
| Инкрементальный ETL + checkpoint | Готово |
| Docker Compose | Готово |
| Учебная KB (~6800 строк) | Готово |
| Аутентификация | Не начато |

**Критерий выхода:** демо для стейкхолдеров; выбрана RAG-конфигурация для пилотного test set.

---

## Фаза 1 — Пилот

**Цель:** Измеримые KPI с реальной KB аэропорта на ограниченной группе (20–50 сотрудников).

| Направление | Результаты |
|-------------|------------|
| **База знаний** | Реальные SOP/FAQ одного сервиса; процесс владельца KB |
| **Качество** | Golden set ≥ 50 вопросов; baseline по [rag_evaluation_ru.md](rag_evaluation_ru.md) |
| **Безопасность** | SSO или API gateway; базовый audit log |
| **Compliance** | Согласование с DPO/юристами; [privacy_ru.md](privacy_ru.md) |
| **Эксплуатация** | Расписание бэкапов; runbook ingest при обновлении KB |
| **Метрики** | Время до ответа, эскалации, удовлетворённость |

**Критерии Go/No-Go** (из PRD):

1. Загружена реальная KB минимум одной зоны
2. Test set прошёл compliance review
3. Реализованы auth и audit
4. Согласована политика обработки данных
5. Назначен владелец KB и процесс обновления
6. Выбрана RAG-конфигурация по A/B
7. План обучения пилотной группы

## Фаза 2 — Масштабирование

**Цель:** Охват всех сервисов аэропорта.

| Функция | Описание |
|---------|----------|
| Полная KB | Все фронтлайн-сервисы |
| Глоссарий (гл. 15) | Включить индексацию при необходимости |
| RBAC | Разделы KB по ролям |
| Обратная связь | Жалобы на ответы → тикеты KB |
| Аналитика | Использование, топ запросов, качество retrieval |
| PostgreSQL | Вместо SQLite |
| Внешний vector DB | При лимитах FAISS |

## Фаза 4 — Продакшен-эксплуатация

**Цель:** Сервис с SLA.

| Возможность | Цель |
|-------------|------|
| HA | Несколько реплик backend |
| Shared SSE | Redis pub/sub |
